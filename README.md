# Human-in-the-Loop AI @ `LangGraph`

This repository demonstrates conversational and agentic workflow orchestration using [LangGraph](https://github.com/langchain-ai/langgraph?utm_source=chatgpt.com) with an emphasis on stateful execution, tool-driven interactions, runtime interruption, and human-in-the-loop control patterns. The implementation progresses from a foundational chatbot pipeline in `1-basicChatBot.ipynb`, where `StateGraph`, message reducers, and graph compilation manage conversational state, into interactive agent workflows in `humanInTheLoop.ipynb` that introduce execution pauses through `interrupt`, human feedback resumption via `Command`, checkpoint-backed memory persistence, and conditional routing using `ToolNode` and `tools_condition`. Runtime execution is powered through [LangChain](https://github.com/langchain-ai/langchain?utm_source=chatgpt.com) abstractions with [Groq](https://groq.com/?utm_source=chatgpt.com) LLM integration (`llama-3.1-8b-instant`) and external augmentation through [Tavily Search](https://github.com/tavily-ai/tavily-python?utm_source=chatgpt.com). The additional `agent.py` module packages a deployable graph definition exposed through `langGraph.json`, enabling local development and visualization through the `langgraph dev --tunnel` workflow and inspection inside [LangGraph Studio](https://github.com/langchain-ai/langgraph?utm_source=chatgpt.com). Execution tracing, debugging, and runtime introspection are further integrated using [LangSmith](https://github.com/langchain-ai/langsmith-sdk?utm_source=chatgpt.com) and reinforced by the `debugging.ipynb` notebook, which demonstrates graph inspection, tool execution paths, and observability practices for iterative development and production-oriented conversational systems.
