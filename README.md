# Human-in-the-Loop AI @ `LangGraph`

This repository implements conversational and agentic workflow orchestration using [LangGraph](https://github.com/langchain-ai/langgraph)⁠![Attachment.tiff](file:///Attachment.tiff) as a stateful execution layer for building interruptible AI systems with human feedback loops. The architecture progresses from foundational graph-driven conversation handling in `1-basicChatBot.ipynb`, where `StateGraph` coordinates message state transitions and execution compilation, into interactive agent execution patterns in `humanInTheLoop.ipynb` that introduce runtime pausing through `interrupt`, resume control via `Command`, checkpoint-backed persistence, and conditional tool routing with `ToolNode` and `tools_condition`. Multi-agent coordination patterns are extended through `multiAgents.ipynb`, while `agent.py` exposes a deployable graph entrypoint registered through `langGraph.json` for runtime execution and local graph serving. Language model execution is integrated through [LangChain](https://github.com/langchain-ai/langchain)⁠![Attachment.tiff](file:///Attachment.tiff) abstractions with [Groq](https://groq.com/)⁠![Attachment.tiff](file:///Attachment.tiff) inference using `llama-3.1-8b-instant`, and external knowledge retrieval is coordinated through [Tavily](https://tavily.com/)⁠![Attachment.tiff](file:///Attachment.tiff). Development and execution workflows support local graph inspection and interactive visualization through LangGraph development tooling, enabling iterative orchestration and deployment workflows. Runtime observability and execution tracing are incorporated using [LangSmith](https://www.langchain.com/langsmith)⁠![Attachment.tiff](file:///Attachment.tiff), with `debugging.ipynb` demonstrating graph inspection, execution tracing, tool invocation analysis, and state introspection to support debugging, evaluation, and production-oriented conversational agent development.
